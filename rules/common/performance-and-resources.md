## 性能与资源使用规则

> 适用范围：与 Agent 调用、工具执行、搜索与文件读写相关的设计和实现。

### 1. LLM 调用与上下文控制

- 避免在单轮对话中附带过多无关上下文：
  - 优先提炼为「摘要 + 关键片段」；
  - 对大文件使用按行读取或部分读取，而非一次性整文件注入。
- 对重复问题，鼓励使用已有会话摘要或结果，而不是从零开始重新分析。

### 2. 搜索与文件读写

- 搜索类工具（如 `search_text`、`glob_search`）必须设置上限：
  - 限制参与搜索的文件数与单文件字节数；
  - 需要大范围扫描时，应在 Plan 中明确说明开销与收益。
- 对大文件的读写应优先采用：
  - 分段读取（按行范围）；
  - 精确定位修改点，而非全文件重写。

### 3. 命令执行与外部资源

- 避免在短时间内重复执行同一高开销命令（如全量测试、重构工具等）；
- 对网络请求与 MCP 调用，应统一控制超时与重试策略，防止阻塞整个 Agent 循环。

### 4. 统计与反馈

- 在需要时，利用 `stats` 或等价统计功能分析：
  - 单次会话的平均耗时与工具调用次数；
  - 最常用或最耗时的操作模式；
- 在 Plan 阶段针对「疑似性能瓶颈」的任务，优先给出优化与折中方案（例如先跑局部测试，再跑全量）。
