## 技能：调试与诊断会话问题

> 适用场景：Agent 行为异常、工具调用频繁失败、响应缓慢或结果质量不佳时，用于系统化定位问题。

### 一、目标

- 通过结构化的诊断流程：
  - 确认配置与环境是否正确；
  - 识别工具或模型调用异常；
  - 提出可执行的修复建议或缓解措施。

### 二、对话提示模版

> 「当前 Agent 行为异常/效果不佳，请作为调试助手执行系统化诊断。  
> 请分步进行：  
> 1) 检查配置与环境（包括 provider/model/workspace/MCP 状态）；  
> 2) 分析近期会话文件与工具调用情况；  
> 3) 给出可能的原因列表；  
> 4) 给出优先级排序的修复建议与验证步骤。」

### 三、典型步骤

1. **环境与配置检查**
   - 建议用户运行 `cai-agent doctor`，检查：
     - LLM provider/model/base_url/api_key；
     - workspace 与配置文件加载路径；
     - MCP 设置与连通性。
2. **会话与工具统计分析**
   - 使用 `sessions` 与 `stats` 子命令查看近期会话：
     - 总耗时、工具调用次数、错误次数；
     - 按模型分布的会话数。
   - 对异常会话文件使用 `load_session` + `_collect_tool_stats` 思路查看：
     - 工具调用顺序、失败频率与最后一个工具。
3. **分类定位问题**
   - 配置/网络类问题：模型列表获取失败、MCP 不可用等；
   - 工具类问题：路径越界、命令不在白名单、搜索超限等；
   - 规划/提示类问题：system prompt 配置不合理或缺少规则/技能指引。
4. **制定修复步骤**
   - 从风险最低、收益最高的修改开始：
     - 调整配置或环境变量；
     - 收紧或放宽工具限制（保持安全前提下）；
     - 优化规则或系统提示。

### 四、注意事项

- 在诊断过程中避免立即大改核心逻辑；优先通过配置、提示与规则调整解决；
- 对于难以复现的问题，应记录出现时的关键信息（配置摘要、会话片段、错误信息），以便后续分析。
