## 技能：安全集成外部 API / 服务

> 适用场景：为项目新增或调整与外部 HTTP API、MCP 服务或其他后端的集成。

### 一、目标

- 在遵守安全与可维护性原则的前提下：
  - 设计清晰的集成边界与适配层；
  - 正确处理认证、超时与错误；
  - 提供可靠的调试与降级方案。

### 二、对话提示模版

> 「请帮助我为项目集成外部 API/服务：\<服务名称或说明\>。  
> 要求：  
> - 明确集成目的与数据流；  
> - 设计适配层接口与错误处理策略；  
> - 遵守安全规则（密钥、超时、重试等）；  
> - 给出调试、验证与降级方案。」

### 三、典型步骤

1. **需求与数据流梳理**
   - 明确输入/输出、调用频率与延迟要求；
   - 标记涉及的敏感字段与安全约束。
2. **接口与适配层设计**
   - 采用统一的 HTTP 客户端封装（如 `httpx`）；
   - 定义清晰的函数/方法签名与返回结构。
3. **错误与重试策略**
   - 对网络波动类错误采用有限重试 + 退避；
   - 对认证失败、参数错误等不做盲目重试；
   - 日志中避免输出完整请求体中的敏感信息。
4. **验证与调试**
   - 设计可重复的调用示例（CLI 或测试）；
   - 提供诊断路径（如何查看请求/响应、如何排查典型错误）。

### 四、注意事项

- 所有密钥与凭证只能来源于环境变量或本地未提交配置；
- 对外部服务的不可用情况，应设计合理的降级行为或友好错误提示。
