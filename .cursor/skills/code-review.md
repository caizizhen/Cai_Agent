## 技能：代码评审（Code Review）

> 适用场景：对某次改动、某个模块或一个 PR 做质量审查，重点关注风险、回归与测试缺口。

### 一、目标

- 以审查者视角识别：
  - 真实 bug 或行为回归；
  - 安全、配置、兼容性风险；
  - 缺失的测试与验证步骤。

### 二、对话提示模版

> 「请以代码评审模式审查这次改动。  
> 重点：  
> - 优先列出 bug、风险、回归点和缺失测试；  
> - 给出证据位置与原因；  
> - 最后再补充简短总结与改进建议。」

### 三、典型步骤

1. 查看改动范围与上下文；
2. 识别高风险模块（配置、工具、命令执行、网络调用等）；
3. 针对行为变化、边界条件和异常路径找问题；
4. 检查是否存在对应的验证或测试；
5. 产出按严重程度排序的 findings。

### 四、注意事项

- 审查重点是发现问题，不是重复描述代码；
- 如未发现明确问题，也应说明残余风险与测试空白。
