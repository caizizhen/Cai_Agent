## Python 配置与环境变量演进规则

> 适用范围：`config.py`、TOML 配置、环境变量覆盖逻辑，以及未来新增配置项。

### 1. 配置新增原则

- 新配置项应先判断是否确有必要：
  - 能用合理默认值解决的，不必强制新增配置；
  - 只有当用户确实需要覆盖行为时，才引入新字段。
- 配置名应尽量稳定、语义清晰，避免短期内频繁重命名。

### 2. 优先级与兼容性

- 保持明确且稳定的优先级：
  - 默认值 < TOML < 环境变量 < CLI 临时覆盖
- 新增配置时，尽量保持向后兼容：
  - 旧配置仍能工作；
  - 若必须废弃旧字段，应在文档和变更日志中说明迁移路径。

### 3. 安全与可见性

- 敏感配置（API Key、Token）只允许来自环境变量或本地私有配置；
- `doctor` 等诊断输出必须打码敏感字段，不得完整回显。

### 4. 文档同步

- 每次新增或调整配置项，都应同步更新：
  - `README.md`
  - 示例 TOML 模板
  - `CHANGELOG.md`
