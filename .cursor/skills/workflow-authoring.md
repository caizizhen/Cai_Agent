## 技能：编写多步 Workflow

> 适用场景：需要把一组固定的多步任务描述成 `workflow` JSON，供 `cai-agent workflow` 顺序执行。

### 一、目标

- 把复杂任务拆分为可复用、可执行、可统计的多步 workflow：
  - 每一步有清晰目标；
  - 便于后续复盘与自动化；
  - 能与 plan、stats、rules 配合使用。

### 二、对话提示模版

> 「请帮我把这个任务设计成 workflow：\<任务描述\>。  
> 请输出每一步的名称、目标、执行顺序、风险点与预期验证方式，并兼顾后续 JSON 化执行。」

### 三、典型步骤

1. 拆分任务为有先后关系的步骤；
2. 给每一步命名，避免模糊标题；
3. 明确每一步的输入、输出和完成判定；
4. 标出哪些步骤适合只规划，哪些步骤适合执行；
5. 给出适合落地到 JSON 的字段建议。

### 四、注意事项

- workflow 的每一步应尽量独立、可理解；
- 避免把多个不相关目标塞进同一步。
