## 技能：规则体系维护与演进

> 适用场景：需要添加、修改或清理 `rules/` 与 `skills/` 中的内容，使其与项目实际状态保持一致。

### 一、目标

- 让规则与技能文档：
  - 覆盖当前项目的主要风险点与最佳实践；
  - 与实现与文档保持同步；
  - 结构清晰，便于 Agent 在 Plan/Execute 阶段引用。

### 二、对话提示模版

> 「请作为规则维护助手，帮助我审查并更新当前的 `rules/` 与 `skills/`。  
> 要求：  
> - 找出过时、重复或缺失的规则/技能；  
> - 给出新增/合并/删除的建议；  
> - 为每个改动提供具体编辑方案与验证方式。」

### 三、典型步骤

1. **现状盘点**
   - 列出当前 `rules/` 与 `skills/` 下所有文件；
   - 按主题归类（安全、性能、测试、工作流等）。
2. **一致性检查**
   - 对照 README 与架构文档，检查：
     - 是否有新能力缺少规则/技能支撑；
     - 是否有文档与实现不一致的地方。
3. **规划改动**
   - 标出需要新增的规则/技能主题；
   - 标记可以合并或简化的文档；
   - 设计编辑顺序，避免一次性大改所有文件。
4. **执行与验证**
   - 按文件逐步修改，保持 diff 清晰；
   - 在 Plan 阶段测试部分新规则/技能是否被正确引用。

### 四、注意事项

- 保持文档风格统一（标题、分节、语言）；
- 避免在规则/技能中出现与安全规则冲突的示例或建议；
- 对高影响规则改动，应在 README 或变更日志中简要说明。
