## Python CLI 与 TUI 设计规则

> 适用范围：`__main__.py`、`tui.py` 以及未来新增的命令行/终端界面相关代码。

### 1. 子命令设计

- 每个子命令应有清晰的职责与帮助信息：
  - 使用 `argparse` 的 `help` 字段说明用途；
  - 避免一个子命令承担太多不相关功能。
- 新增子命令时需评估：
  - 是否可以复用现有命令（如通过参数扩展）；
  - 是否会与现有命令语义冲突。

### 2. 参数与默认值

- 必填参数应在帮助信息中明确标出；
- 对影响行为的重要参数（如 `--json`、`--workspace`、`--model`）：
  - 应在文档中说明优先级与默认行为；
  - 在实现中保持与配置加载逻辑一致。

### 3. 交互体验

- CLI 模式下：
  - 普通模式输出人类可读文本；
  - `--json` 模式仅输出单行 JSON，避免混入日志。
- TUI 模式下：
  - 对长输出进行分页或滚动处理；
  - 对错误与告警使用明显但不过度刺眼的样式。
