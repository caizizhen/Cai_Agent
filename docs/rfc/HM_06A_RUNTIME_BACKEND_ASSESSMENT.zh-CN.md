# HM-06a：Runtime backend 产品化评估（结论文档）

> 与 **`CLOUD_RUNTIME_OOS.zh-CN.md`** 一致：**默认云后端 OOS**；本文件只排 **本地 / 容器 / SSH** 的交付优先级。

## 1. 当前能力基线

- **`[runtime]`** 与 **`get_runtime_backend`**：已支持 **`local`** 及若干占位后端字段（Docker/SSH/Modal/Daytona 等）。
- **工具层 `run_command`**：以 **`runtime_backend`** 解析执行路径；产品化差距主要在 **文档、默认值、故障自检** 与 **doctor 暴露度**。

## 2. 优先级（产品化实现顺序建议）

| 顺序 | 后端 | 说明 |
|------|------|------|
| 1 | **`local`** | 默认且最完整；持续保证 **Windows / POSIX** 行为一致与安全策略 |
| 2 | **`docker`** | 团队 CI / 可复现构建首选；需明确 **镜像、卷挂载、资源限制** 与 **doctor** 诊断 |
| 3 | **`ssh`** | 远程开发/跳板机场景；需 **known_hosts、密钥路径、超时** 与审计对齐 |
| 4 | **Modal / Daytona / Singularity…** | **条件立项**；默认 **文档指向 OOS** 与替代方案（本地/自建 runner） |

## 3. 风险与边界

- **云休眠省钱**：涉及计费账号与数据出境，**不纳入默认开源交付承诺**。
- **多后端自动切换**：属于 **P2** 编排，应晚于 **单一后端可靠性**。

## 4. 验收

- 路线图可将 **`HM-06`** 后续实现拆为：**docker 产品化** → **ssh 产品化** → **云条件单独立项**。
