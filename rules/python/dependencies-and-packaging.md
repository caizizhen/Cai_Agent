## Python 依赖与打包规则

> 适用范围：与 `pyproject.toml`、依赖管理、打包与发布相关的改动。

### 1. 依赖管理

- 新增依赖前应先评估：
  - 是否可以通过标准库或现有依赖实现；
  - 引入依赖的维护成本与安全风险。
- 在 `pyproject.toml` 中：
  - 统一使用兼容区间（例如 `>=x.y`），避免过度锁死版本；
  - 对核心依赖注明用途（在文档或注释里）。

### 2. 打包与发布

- 保持包结构与 `where = ["src"]` 等配置一致；
- 新增模板或数据文件时，确保 `package-data` 中已正确包含；
- 不将构建产物（如 `*.egg-info`、`dist/` 等）提交到仓库。

### 3. 本地开发与可重复环境

- 推荐使用虚拟环境或工具（如 venv、poetry 等）隔离依赖；
- 在文档中提供清晰的「本地安装与开发」指引，例如：
  - `pip install -e .`
  - 必要的环境变量与前置软件。
