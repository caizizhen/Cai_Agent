## Python 风格与类型标注规则

> 适用范围：本仓库内所有 Python 代码；Agent 在生成或修改 Python 代码时都应优先遵守本规则。

### 1. 基本代码风格

- 遵循 PEP 8 基本约定：
  - 缩进使用 4 个空格；
  - 行宽建议不超过 100 列，超过时适当换行；
  - 模块按顺序分组导入：标准库 → 第三方 → 本地包。
- 禁止使用通配符导入（`from x import *`），应显式列出需要的名称。

### 2. 类型标注

- 新增的函数与方法必须添加类型标注，包括：
  - 参数类型与返回类型；
  - 如无法精确标注，可使用 `str | None`、`dict[str, object]` 等合理宽泛类型，而不是完全省略。
- 面向外部调用的 API（如工具函数、配置加载函数）优先保证类型标注完整且清晰。

### 3. 异常与返回值

- 函数不要既通过返回值又通过异常表达同一类错误：
  - 预期错误：返回结构化结果（如 `Ok/Err` 风格或带 `success` 字段）或抛出自定义异常；
  - 非预期错误：允许直接抛出异常，由上层统一处理。
- 内部工具函数如返回 `None` 代表失败，必须在文档或函数名中明确表达（例如 `try_` 前缀或注释说明）。

### 4. 异步与同步

- Agent 与工具当前以同步为主；在需要引入异步代码时应遵守：
  - 单个模块内尽量保持「要么同步、要么异步」的一致性；
  - 异步函数命名以 `async def` + 动词开头，并在名称上不额外加 `async_` 前缀。

### 5. 文档与注释

- 注释只用于解释非显而易见的设计意图、约束或权衡，避免简单「翻译代码」式注释；
- 对外公开的函数/类应提供 docstring，简要说明用途与关键参数。
