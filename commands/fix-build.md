# /fix-build

## 用途

针对构建失败、测试失败、类型检查失败等问题，进行最小修复闭环：

1. 识别根因
2. 给出最小修改方案
3. 执行验证步骤
4. 输出修复结果与剩余风险

## 输入建议

- 报错日志（优先完整堆栈）
- 复现命令（如 `pytest -q`、`npm run build`）
- 影响范围（目录/模块）

## 输出要求

- 根因分析（按优先级）
- 修复步骤（可执行）
- 验证结果（通过/失败 + 证据）
- 未解决项（若有）

## 与质量门禁联动

- CLI `cai-agent fix-build` 默认会在修复后执行一次 `quality-gate`。
- 若仅希望生成修复方案而不触发门禁，可使用 `--no-gate`。
