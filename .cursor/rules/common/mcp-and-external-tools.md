## MCP 与外部工具接入规则

> 适用范围：MCP Bridge、第三方工具服务、外部命令封装与未来新增的桥接能力。

### 1. 接入边界

- 外部工具只负责补充能力，不应破坏本地 Agent 的安全边界；
- 外部工具的调用入口必须统一收敛，避免在多个模块中散落重复实现。

### 2. 输入与输出约束

- 传给外部工具的参数必须显式构造并做基础校验；
- 对外部工具返回内容：
  - 默认视为不完全可信；
  - 对关键字段做存在性与类型检查；
  - 不直接执行其返回的任意命令或脚本内容。

### 3. 可用性与超时

- 外部服务不可用时，应给出清晰的错误提示与降级方案；
- 所有调用都应设置超时，并限制重试次数，避免卡死整个 Agent 流程。

### 4. 文档与运维

- 新增 MCP 或外部工具能力时，应同步补充：
  - 配置项说明；
  - 典型用法；
  - 探活与诊断方式（例如 `mcp-check`）。
