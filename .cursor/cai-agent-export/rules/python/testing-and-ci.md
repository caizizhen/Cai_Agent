## Python 测试与验证规则

> 适用范围：本仓库内所有 Python 模块；Agent 在规划或实现改动时需考虑相应的验证策略。

### 1. 测试优先级

- 优先为下列代码补充或维护测试：
  - 配置解析与环境变量逻辑；
  - 工具系统（文件操作、搜索、命令执行等）；
  - 会话持久化与统计逻辑（如 `sessions`、`stats`）。
- 对复杂或高风险逻辑（如安全相关校验）应尽量编写单元测试或集成测试。

### 2. 测试风格

- 建议使用 pytest 风格测试（函数式、断言语句简洁）；
- 测试函数命名以 `test_` 开头，表达清晰场景，例如：
  - `test_resolve_workspace_path_blocks_parent_directory`;
  - `test_run_command_rejects_shell_metacharacters`。

### 3. Agent 改动的验证建议

- 在 `plan` 阶段，针对每个改动建议至少给出一种验证方式，例如：
  - 运行某个 CLI 命令（如 `cai-agent doctor` 或 `cai-agent run --json`）；
  - 运行特定测试模块或测试用例；
  - 手工验证步骤（如「在 TUI 中尝试某个斜杠命令」）。
- 在执行阶段，生成的计划应尽量避免「完全不验证」的改动。

### 4. CI 与本地一致性

- 即便当前仓库未配置 CI，Agent 也应假设未来可能接入 CI：
  - 生成的脚本与命令尽量使用相对路径与显式依赖；
  - 避免依赖本地环境的隐式状态（如未声明的环境变量）。
