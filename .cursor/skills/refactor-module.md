## 技能：安全重构单个模块

> 适用场景：对某个现有模块进行结构优化、命名调整、逻辑拆分等「不改变对外行为」的改动。

### 一、目标

- 在不改变模块对外 API 与语义的前提下：
  - 简化内部结构、消除重复代码；
  - 提升可读性与可维护性；
  - 降低未来改动的风险。

### 二、对话提示模版

> 「请作为重构助手，帮助我安全重构模块：\<模块路径或名称\>。  
> 要求：  
> - 不改变对外可见行为与接口签名（除非另有说明）；  
> - 遵守本仓库的规则（命名、日志、安全、Python 风格等）；  
> - 给出分步计划：每步改动目标、涉及文件、工具使用与验证方式；  
> - 每步结束后使用 `git diff` 或等效手段确认影响范围。」

### 三、典型步骤

1. **分析与定位**
   - 使用 `search_text` / `glob_search` 找出模块文件及其引用位置；
   - 概要说明当前结构（主要类/函数、依赖关系）。
2. **识别问题与改进点**
   - 找出长函数、重复逻辑、过深嵌套等问题；
   - 标记潜在风险点（如跨模块隐式状态、全局变量等）。
3. **设计重构方案**
   - 拆分为若干小步，每一步尽量保持可回滚：
     - 提取函数/类；
     - 调整命名与参数；
     - 拆分文件或移动代码。
4. **执行与验证**
   - 每一步执行后：
     - 使用 `git diff` 查看改动范围是否符合预期；
     - 给出建议的测试或验证命令（如运行某个 CLI 或测试）。

### 四、注意事项

- 避免在一次提交中同时进行大量无关的重构（如命名大改 + 行为修改）；
- 对外部依赖和公共 API 的改动需额外审慎，最好单独一轮计划与执行；
- 重构前后尽量保持日志与错误处理策略一致，仅在有明确收益时再调整。
