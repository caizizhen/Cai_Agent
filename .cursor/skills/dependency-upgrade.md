## 技能：依赖升级与兼容性检查

> 适用场景：升级 Python 依赖、替换底层库或调整运行时版本时，评估影响并制定安全升级方案。

### 一、目标

- 安全地升级依赖或运行时：
  - 明确升级动机与预期收益；
  - 分析可能的兼容性风险；
  - 设计验证与回滚路径。

### 二、对话提示模版

> 「请帮助我升级/调整以下依赖或运行时：\<列出依赖与版本变化\>。  
> 要求：  
> - 说明升级原因与潜在风险；  
> - 罗列受影响的模块与调用点；  
> - 设计分步升级计划与验证策略；  
> - 给出失败时的回滚方案。」

### 三、典型步骤

1. **收集信息**
   - 查看当前 `pyproject.toml` 或相关配置；
   - 使用搜索工具查找依赖相关的导入与调用。
2. **评估影响范围**
   - 按模块或功能分组受影响代码；
   - 标记对外 API 或行为有变化的部分。
3. **设计升级计划**
   - 优先从影响较小的模块着手；
   - 如果可能，支持「灰度」或可配置回退。
4. **验证与回滚**
   - 逐步运行测试或关键 CLI 命令；
   - 保持升级前后的 diff 清晰可审；
   - 预先准备回滚命令或配置（如恢复版本号、还原锁文件等）。

### 四、注意事项

- 对安全性敏感的依赖（如 HTTP 客户端、加解密库）需特别审慎；
- 升级过程中如遇到大面积 API 变化，应考虑分阶段迁移或寻找替代库。
