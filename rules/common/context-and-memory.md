## 上下文与记忆管理规则

> 适用范围：Agent 的系统提示、会话恢复、项目上下文注入、未来的记忆/摘要能力。

### 1. 上下文最小化原则

- 只注入解决当前任务真正需要的上下文：
  - 优先使用摘要、目录信息与关键代码片段；
  - 避免把整份长文档或大文件无差别塞进提示词。
- 对重复出现的项目背景，优先沉淀为规则、技能或短摘要，而不是每次重新拼接大段文本。

### 2. 项目上下文来源

- 项目级说明优先来源于：
  - `README.md`
  - `docs/ARCHITECTURE.zh-CN.md`
  - 未来专门的项目说明文件（如 `CAI.md` / `AGENTS.md`）
- 引入上下文时应说明其来源，并注意避免过期文档误导模型。

### 3. 会话与记忆边界

- 会话历史仅作为当前任务的辅助上下文，不应盲目继承所有旧状态；
- 从旧会话恢复时，应优先保留：
  - 当前目标相关的消息；
  - 上轮生成的计划、关键工具结果与风险说明；
  - 不必要的噪音对话应考虑裁剪或摘要化。

### 4. 记忆的可信度

- 任何「记忆」或历史总结都不是绝对事实：
  - 当记忆与当前代码/文档冲突时，以当前代码与文档为准；
  - 重要结论必须能回溯到文件、配置或工具输出。
