## 技能：性能与开销评估（Performance Review）

> 适用场景：Agent 使用中出现响应缓慢、工具调用过多、token 开销过大等情况，希望评估并优化整体性能。

### 一、目标

- 基于已有统计与会话记录：
  - 分析延迟与资源开销的主要来源；
  - 识别不必要或低效的工具调用与 LLM 交互；
  - 给出具体的优化建议与权衡。

### 二、对话提示模版

> 「请对近期 Agent 使用情况做一次性能评估。  
> 要求：  
> - 基于 `stats` 与会话文件分析耗时与工具调用开销；  
> - 找出最耗时/最频繁的操作模式；  
> - 提出可以落地的优化建议（如缩短上下文、减少不必要工具调用等）；  
> - 区分短期可做与长期可做的优化。」

### 三、典型步骤

1. **收集数据**
   - 建议运行 `cai-agent stats`，查看：
     - 总会话数、总/平均耗时；
     - 总/平均工具调用次数与错误数；
     - 不同模型的使用分布。
   - 选择若干代表性会话文件进行深入分析。
2. **分析模式**
   - 对慢会话或工具调用密集的会话，查看：
     - 是否存在重复或无效的工具调用回路；
     - 是否在同一个问题上进行了多轮冗长交互；
     - 是否频繁重读大文件或重复搜索。
3. **提出优化方案**
   - 提示层面：
     - 优化 system prompt 或规则，使 Agent 更精简地规划与调用工具；
     - 在 Plan 阶段增加「限制工具调用轮数/范围」的自约束。
   - 工具层面：
     - 为常用但昂贵的操作提供缓存或简化模式；
     - 对搜索类工具增加更严格的上限（文件数、字节数等）。
4. **验证与回归**
   - 在应用优化后，选择部分典型任务再次运行；
   - 对比 `stats` 输出，观察平均耗时与工具调用次数是否有明显变化。

### 四、注意事项

- 性能优化应在不牺牲安全性与正确性的前提下进行；
- 在缺乏充分数据时，避免做过早或过度复杂的优化；
- 优化建议应结合具体项目规模与用户需求灵活取舍。
