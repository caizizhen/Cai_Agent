## 安全与敏感信息处理规则

> 适用范围：本仓库内所有与文件、配置、网络、命令执行相关的代码与改动。

### 1. 敏感信息与配置

- **绝不提交真实密钥**：
  - API Key、Token、密码、私钥等只能通过环境变量或本地未提交配置文件（如 `.env.local`）传入；
  - 示例配置文件（如 `cai-agent.example.toml`）中仅使用占位符。
- **配置文件中的隐私字段**：
  - 在文档与输出中，只使用示例值或打码值；
  - Agent 不得建议用户将真实 token 写入版本库。

### 2. 文件与路径访问

- 所有文件操作必须通过「工作区沙箱」路径解析，禁止：
  - 使用绝对路径访问系统目录；
  - 通过 `..` 越界访问工作区外部；
  - 修改诸如 SSH 配置、系统 Hosts 等敏感文件。
- 对于用户提供的路径参数，必须做必要的校验（例如禁止包含 `..` 或非预期前缀）。

### 3. 命令执行安全

- 仅允许执行白名单中的命令（如 `git`, `pytest` 等），禁止：
  - 通过路径（`./script.sh`、`/usr/bin/*` 等）执行；
  - 使用 `;`、`&&`、`|`、重定向等 shell 组合语法；
  - 执行明显破坏性命令（如 `rm -rf`、`mkfs` 等）。
- 对 `cwd` 进行严格限制：必须位于工作区内，不得指向用户 home 或系统目录。

### 4. 外部服务与 MCP 工具

- 仅在明确配置并启用的前提下调用 MCP 或其他外部工具；
- 对外部工具返回的数据不要盲信：
  - 需要对关键字段进行基本校验；
  - 对生成代码/命令的内容仍需按本规则做安全检查。

### 5. Agent 行为边界

- 本 Agent 的设计目标是「代码理解与小规模改动」，不应：
  - 主动创建系统级用户、修改系统权限；
  - 大规模重写非当前项目的代码。
- 在计划或执行中，如某项操作存在潜在安全风险，Agent 应主动在输出中注明，并给出更安全的替代方案或手动步骤建议。
