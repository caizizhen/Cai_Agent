## 技能：新增功能并配套测试

> 适用场景：在现有项目中增加新的 CLI 子命令、工具函数或 Agent 能力，同时补充或更新测试。

### 一、目标

- 为现有系统新增一项具体功能，同时：
  - 明确需求与边界条件；
  - 设计合理的模块与接口；
  - 为关键路径增加测试或至少给出验证方案。

### 二、对话提示模版

> 「请帮我在项目中新增功能：\<功能描述\>。  
> 要求：  
> - 遵守本仓库的命名、风格、安全与测试规则；  
> - 先给出 Plan-Then-Execute 风格的实施方案；  
> - 为关键逻辑设计至少一种自动测试或验证方法；  
> - 每个步骤说明将创建/修改的文件与使用的工具。」

### 三、典型步骤

1. **需求澄清**
   - 明确输入、输出、边界条件与失败场景；
   - 标记是否涉及外部服务或敏感信息。
2. **查找现有基础设施**
   - 使用 `search_text` / `list_tree` 寻找类似功能或可复用模块；
   - 评估是复用现有模式，还是需要新增模块。
3. **设计接口与数据结构**
   - 定义新的函数/类/CLI 子命令的签名与行为；
   - 明确与现有模块的依赖关系。
4. **实现步骤**
   - 按顺序实现最小可行路径：
     - 新增文件或扩展现有模块；
     - 接线到 CLI/Graph/工具系统等；
     - 保持每步改动可通过 `git diff` 清晰审查。
5. **测试与验证**
   - 设计并（尽可能）实现测试用例，或至少给出精确的手工验证步骤；
   - 建议运行的命令示例，例如：`pytest path/to/test_*.py` 或调用某个 CLI 命令。

### 四、注意事项

- 避免在新增功能的同时大规模重构不相关代码；
- 对错误处理与日志策略保持与现有代码一致；
- 如果涉及配置项变更（例如 TOML 新字段），需要更新相应文档与示例配置。
