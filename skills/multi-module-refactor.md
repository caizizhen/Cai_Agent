## 技能：多模块协同重构

> 适用场景：一次性调整多个相互依赖模块的结构（例如拆分大模块、统一命名、抽取共享工具）。

### 一、目标

- 在控制风险的前提下，对多个模块进行协同重构：
  - 消除重复逻辑与交叉依赖；
  - 统一公共抽象与命名；
  - 保持对外行为整体兼容。

### 二、对话提示模版

> 「请作为多模块重构助手，帮助我重构以下模块：\<列出模块或目录\>。  
> 要求：  
> - 先给出多模块视角的整体结构图与问题清单；  
> - 按步骤规划重构顺序（每步影响尽量局部）；  
> - 每步说明涉及的文件、重构方式、工具调用与验证方法；  
> - 完成后给出新的模块关系与迁移注意事项。」

### 三、典型步骤

1. **全局结构分析**
   - 使用 `list_tree` 与 `search_text` 罗列相关模块与依赖关系；
   - 标记重复逻辑、循环依赖与过度耦合部分。
2. **设计目标结构**
   - 绘制或用文字描述期望的新结构；
   - 指定哪些逻辑需要迁移到共享工具模块。
3. **分步实施计划**
   - 按照「先抽取共享依赖 → 再调整边缘模块 → 最后调整核心模块」的顺序拆分步骤；
   - 每步规划所需工具调用（如搜索/写文件/git diff）。
4. **执行与验证**
   - 严格按计划操作，每步结束都：
     - 检查 `git diff`；
     - 运行必要的测试或 CLI 验证命令。

### 四、注意事项

- 尽量避免在同一步中既改变行为又大改结构；
- 对公共 API 的修改需特别标注，并在文档中同步更新；
- 建议在完成一轮重大重构后进行一次安全扫描与性能评估。
