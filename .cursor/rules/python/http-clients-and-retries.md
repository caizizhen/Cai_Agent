## Python HTTP 客户端与重试规则

> 适用范围：`httpx` 或其他 HTTP 客户端调用，包含 LLM API 与 MCP/外部服务调用。

### 1. 超时必填

- 每个请求必须设置明确超时，不依赖无限默认值；
- 长耗时请求应区分连接超时与读取超时策略。

### 2. 重试策略

- 仅对可重试错误（如 429/502/503/504、短暂网络错误）进行有限重试；
- 对认证失败、参数错误等不可重试错误应快速失败。

### 3. 幂等性

- 可能重复请求的操作要确认幂等性；
- 非幂等接口若需重试，应有去重或保护机制。

### 4. 可观测性与隐私

- 记录请求耗时、状态码与失败类别；
- 不记录完整敏感头、token、密钥和用户隐私负载。
